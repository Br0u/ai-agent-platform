## 部署步骤

<Steps>

### 配置环境变量

至少配置 `DATABASE_URL`、`BETTER_AUTH_SECRET` 和 `ASSISTANT_PUBLIC_ORIGIN`。密钥只放在服务器环境或密钥管理服务中。

### 初始化数据库

执行迁移或初始化脚本，确认账号、角色和权限表已创建，再启动 Web 服务。

### 验证健康状态

检查 `/api/health/live` 和 `/api/health/ready`，然后用普通客户、员工和管理员账号分别验证访问边界。

</Steps>

<Callout type="warning">
  不要把 `.env`、数据库密码、初始管理员密码或 TOTP 恢复码提交到 Git 仓库。
</Callout>
