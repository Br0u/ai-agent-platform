# 助手状态灯颜色设计

## 目标

为悬浮聊天窗口和 `/assistant` 全页工作区的服务状态灯增加一致的语义颜色，同时保持现有尺寸、布局、文案和无障碍状态播报不变。

## 方案

复用两个状态节点已有的 `data-capability` 属性，在 `floating-chat-widget-shadcnui.css` 和 `assistant-workspace.css` 中增加相同的状态选择器，不向 React 组件添加颜色判断。

- `placeholder`：黄色 `#b38225`，表示模型未配置或等待服务。
- `available`：绿色 `#27826b`，表示服务已就绪。
- `degraded`：红色 `#b94b5a`，表示基础服务暂不可用。
- 每种状态的外圈光晕使用同色低透明度，保持当前 7px 圆点和 3px 光晕尺寸。

默认样式对应 `placeholder`。`available` 和 `degraded` 通过属性选择器覆盖圆点及光晕颜色。状态数据仍由现有服务状态响应驱动，组件结构、请求流程和错误处理均不改变。

## 验证

- 在两个组件测试中检查三种 `data-capability` 状态仍正确渲染，并检查 CSS 包含三种状态的颜色映射。
- 运行悬浮聊天组件、全页工作区组件测试、类型检查和 lint。
- 在浏览器中确认悬浮窗与 `/assistant` 的就绪状态均显示绿色，且标题区域尺寸和对齐未发生变化。

## 范围外

- 不修改停靠助手的状态灯。
- 不调整状态文案、刷新逻辑、动画和对比度之外的视觉样式。
