# 码多多 Hero 莫比乌斯带视觉升级设计

## 目标

将 `/product/code-agent` 首屏右侧的当前 CSS 方框叠加与文字胶囊，替换为真正的莫比乌斯带三维视觉。视觉重点是先保证几何正确，再叠加现有页面的蓝紫科技质感。

## 已确认的视觉方向

- 使用单次 180° 半扭转的带状曲面，不使用普通圆环、两个椭圆叠加或伪造的双环。
- 带面应有明确宽度、连续外轮廓和可见的前后遮挡关系；旋转时能看出同一条带面在半扭转处翻转。
- 颜色延续当前页面的蓝、紫、青渐变，但渐变附着在真实曲面上；保留底部柔和光晕，删除当前三个方框和三条文字丝带。
- 动画以缓慢环绕旋转为主，叠加轻微上下浮动；不做高频闪烁或大幅位移，避免抢过左侧标题。

## 实现边界

### 组件

新增一个客户端 `MobiusStripVisual` 组件，负责：

1. 根据标准参数化表面生成固定分辨率的网格：
   - `x = (R + v cos(u / 2)) cos(u)`
   - `y = (R + v cos(u / 2)) sin(u)`
   - `z = v sin(u / 2)`
   - `u` 覆盖 `0..2π`，`v` 覆盖带宽范围。
2. 在 Canvas 2D 中绘制带面三角形，并按深度排序，保证旋转时遮挡关系正确。
3. 给不同曲面区域施加蓝紫渐变和柔和高光，绘制两条外边界和一条低对比度中心线，用于强化半扭转结构。
4. 使用 `requestAnimationFrame` 驱动持续动画；组件卸载时取消动画帧。

### 页面接入

- `apps/web/src/app/product/code-agent/page.tsx` 仅保留 Hero 的布局结构，将 `.ca-3d-cube-group` 与 `.ca-ribbon*` 替换为 `MobiusStripVisual`。
- `apps/web/src/app/product/code-agent/code-agent.css` 负责容器尺寸、底部光晕、响应式布局和降级状态；不再维护伪 3D 方框/文字丝带样式。

## 动画与可访问性

- 默认动画：约 12–16 秒完成一次完整视角循环，并带有幅度很小的上下浮动。
- `prefers-reduced-motion: reduce` 下停止连续动画，显示静态但完整的莫比乌斯带首帧。
- Canvas 提供 `aria-label`，并保留可读的隐藏文本说明“带有一次 180° 半扭转的莫比乌斯带”。
- 移动端缩小 Canvas 高度和网格分辨率，保持页面加载和滚动流畅。

## 不做的事情

- 不引入 Three.js 或新的运行时依赖。
- 不把莫比乌斯带做成页面主交互，不添加鼠标拖拽、点击弹窗或额外说明面板。
- 不修改首页其它区域、导航、文案和页面路由。

## 验证标准

- `/product/code-agent` 可正常渲染，类型检查、Lint 和已有测试通过。
- 桌面端能明确看出带宽和半扭转，不会被误认为普通环或双椭圆。
- 动画连续、平滑，不出现闪烁、跳帧、画布溢出或卸载后仍运行的动画帧。
- 开启减少动效后保持静态可见；移动端不发生横向溢出。
