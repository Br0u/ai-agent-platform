# AI 助理 Prompt Input 设计规格

状态：已确认，2026-07-15

## 目标

将用户提供的 PromptInput 参考中的弹性输入框、附件缩略图和工具栏交互融入现有 AI 助理，同时保持现有文本会话、500 字限制、服务状态、预设问题和路由行为不变。

## 视觉与交互

- 紧凑态输入框在未聚焦时保持低高度；聚焦或输入后平滑展开。
- 文本区域随内容自动增高，达到上限后内部滚动。
- 发送按钮保持当前会话语义，只负责发送文本，不引入新的取消请求行为。
- 输入区使用淡蓝靛紫渐变、圆润玻璃表面、低强度辉光和至少 44px 的交互命中区。
- 浮窗和 `/assistant` 全屏页使用同一个 PromptInput 组件，避免两套输入体验继续分叉。

## 附件范围

首期支持图片附件：最多 6 个，使用本地 object URL 预览、删除和放大预览。当前 `/api/v1/assistant/chat` 只接受 JSON 文本请求，未接入多模态模型前不上传或静默丢弃图片；选择附件后保留缩略图，并明确提示“等待多模态模型接入”。

## 未来入口

- 模型入口显示当前默认的“AI 助理”，点击可打开玻璃菜单；未配置模型显示“即将开放”，不改变当前请求。
- 语音入口保留麦克风按钮，点击只显示“语音输入即将开放”，不申请麦克风权限、不模拟录音。

## 组件边界

- `AssistantPromptInput` 负责输入状态、附件生命周期、模型菜单、语音占位提示、自动高度和提交按钮。
- `AssistantConversation` 负责消息记录、文本提交和服务反馈，将当前会话接入 PromptInput。
- `FloatingChatWidget` 负责浮窗身份、预设问题和路由入口，将快速会话接入同一个 PromptInput。
- 服务端请求契约本轮不改；未来多模态 API 确认后再单独扩展传输协议。

## 验收标准

1. 浮窗和全屏页输入框都能展开、自动增高、发送文本并保留原有键盘行为。
2. 图片可选择、预览、删除和查看大图；最多 6 个，非图片或超限文件给出明确提示。
3. 有附件时不会把图片假装发送给当前文本 API，附件不会被静默清除。
4. 模型菜单和语音按钮可访问、可操作，但未配置能力不会伪装成可用能力。
5. `FloatingChatWidget`、`/assistant`、缩小返回主页面、CONVERSATIONS 和现有助手测试全部保持通过。
