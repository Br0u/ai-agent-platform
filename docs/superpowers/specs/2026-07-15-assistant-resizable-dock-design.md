# AI 助手可伸缩右侧工作区设计

> 日期：2026-07-15<br>
> 状态：已确认<br>
> 视觉方向：B 型覆盖式右侧抽屉 + A 型完整聊天内容<br>
> 范围：公开门户的快速助手、右侧工作区与 `/assistant` 全页工作区

## 背景

当前站点已有两个合理入口：页面右下角的快速助手，以及 `/assistant` 完整工作区。两者不是重复实现，而是不同使用深度的入口。当前缺口是中间态：用户希望在不离开当前页面的情况下展开一个更大的助手工作区，并能按任务需要调整宽度。

本次新增覆盖式右侧工作区，参考企业云产品的侧边 AI 助手，但不复制其品牌、内容或界面。现有快速助手和完整工作区继续保留，三种形态共享同一会话状态。

## 目标

1. 在当前页面右侧打开可伸缩的 AI 助手工作区，不挤压或重排底层页面。
2. 保留快速助手与 `/assistant` 完整工作区，形成“快速问答 → 侧边工作区 → 全页工作区”的连续体验。
3. 三种形态共享消息、输入草稿、服务状态和会话，不因切换形态丢失内容。
4. 桌面端支持指针拖拽和键盘调整宽度，并在后续访问中记住用户宽度。
5. 移动端使用全屏工作区，避免窄屏拖拽与不可用的双栏布局。
6. 保持现有 BFF、AgentOS、限流、错误合同和占位模式不变。

## 非目标

- 不接入新的模型、Skill、会话持久化或计费能力。
- 不实现多会话列表、会话重命名或云端历史同步。
- 不删除快速浮窗，也不把 `/assistant` 全页工作区改成同一套布局。
- 不改变服务端助手 API、权限规则、限流和降级语义。
- 不制造可用性假象；未接入的“新建会话”等能力保持禁用或不展示。

## 体验模型

### 三种形态

| 形态 | 用途 | 打开方式 | 关闭/切换 |
| --- | --- | --- | --- |
| 快速助手 | 简短问答 | 右下角 M 入口 | 关闭，或展开为右侧工作区 |
| 右侧工作区 | 边看页面边连续对话 | 快速助手“展开”、页面助手入口 | 收起为快速助手、关闭、进入全页 |
| 全页工作区 | 长时间专注对话 | 右侧工作区“全屏”、直接访问 `/assistant` | 浏览器导航返回 |

右侧工作区采用覆盖式抽屉，不改变底层页面宽度。打开时显示低对比度遮罩，底层页面保留上下文但不接受交互。

同一时刻只能挂载一个可交互的助手界面：

| 当前路由 | `surface` | 实际挂载界面 |
| --- | --- | --- |
| 非 `/assistant` | `closed` | 无 |
| 非 `/assistant` | `quick` | 快速助手 |
| 非 `/assistant` | `dock` | 右侧工作区 |
| `/assistant` | 任意输入状态 | 仅全页工作区；Provider 将 `surface` 规范化为 `closed` |

进入 `/assistant` 时必须卸载 quick/dock，返回门户时保持 `closed`，避免同时存在两个 dialog、两个输入框或两个焦点目标。composer 只由当前唯一挂载的界面注册。

### 打开与关闭

- 从快速助手展开时，当前消息、草稿和服务状态原样进入右侧工作区。
- 从右侧工作区进入 `/assistant` 时，使用同一 `AssistantExperienceProvider`，不得重建会话。
- 门户顶部“AI 助理”入口打开右侧工作区；右下角 M 入口仍打开快速助手；`/assistant` 通过右侧工作区的“完整工作区”操作或直接 URL 进入。
- 点击遮罩或按 `Esc` 关闭右侧工作区，并将焦点还给打开它的触发器。
- “收起”切回快速助手；“关闭”结束当前展示，但不清空会话。
- 路由切换时由站点壳保持会话，右侧工作区关闭，避免覆盖新页面。
- 发送中的请求在 quick、dock 与 `/assistant` 之间切换时继续执行；请求沿用发起瞬间的 pathname 作为上下文，响应写回共享会话。仅 Provider 卸载、超时、端点变化或显式取消时中止请求。路由变化本身不得重置 `requestStatus`。

## 桌面布局

### 尺寸

- 默认宽度：`480px`。
- 最小宽度：`380px`。
- 最大宽度：`760px`，同时不得超过当前视口减去 `48px` 的安全边距。
- 抽屉占满可视区域高度，固定在右侧，层级高于站点导航和快速助手。
- 左侧调整手柄视觉宽度约 `1px`，可命中区域至少 `6px`。

### 宽度调整

- 指针按下手柄后使用 pointer capture，拖动实时更新宽度。
- 宽度必须在最小值、最大值和当前视口上限之间钳制。
- 手柄可获得键盘焦点：`ArrowLeft` 增宽、`ArrowRight` 缩窄，每次 `16px`；按住 `Shift` 时每次 `48px`。
- 手柄使用 `role="separator"`、垂直方向和当前/最小/最大值语义。
- 调整结束后写入 `localStorage`；存储键版本化，例如 `ai-agent-platform:assistant-dock-width:v1`。
- `pointerup`、`pointercancel`、`lostpointercapture`、窗口失焦和组件卸载都必须结束拖拽，恢复 `user-select`、cursor、pointer capture 与事件监听器。
- 视口缩小时只钳制本次渲染宽度，不覆盖用户保存的偏好宽度；视口恢复后继续使用原偏好值。用户主动拖拽或键盘调整时才保存新值。
- 服务端渲染阶段不读取浏览器存储。key 的版本后缀定义格式版本；旧 key 不读取。空值、非数字、非有限数和超范围值回退并钳制为默认值。
- `localStorage` 的读取、解析和写入全部用异常边界保护。隐私模式、配额不足或存储不可用时仍可调整当前宽度，只是不持久化，不向用户抛出运行时错误。

## 移动端

- `720px` 及以下改为全屏覆盖，不显示拖拽手柄。
- 面板宽高均使用动态视口单位，适配移动浏览器地址栏。
- 头部保留收起/关闭和进入全页入口；触控目标不小于 `44×44px`。
- 输入区处理安全区 `env(safe-area-inset-bottom)`，不能被键盘或系统区域遮挡。
- 软键盘导致可视视口变小时，工作区使用新的动态视口高度，消息区独立滚动，输入区固定在工作区底部；输入框获得焦点时将输入区与最后一条消息滚入可视区域。键盘收起后保持用户原消息滚动位置或底部锚点，不产生整页跳动。
- 窗口从桌面跨到移动断点时立即结束拖拽并隐藏手柄，但不改写已保存宽度；返回桌面后重新按偏好值和当前视口钳制。

## 视觉系统

右侧工作区外层采用 B 型克制企业面板，内部采用 A 型完整聊天体验。

### 外层

- 背景为接近实色的暖白，不使用高透明玻璃导致正文穿透。
- 蓝、靛、紫仅用于状态、焦点、主按钮和极轻的环境光，不铺大面积高饱和渐变。
- 左侧仅保留细手柄与轻阴影，避免厚重双边框。
- 顶部高度约 `56px`；标题、服务状态和操作按钮清晰分区。
- 图标按钮无常驻外框，悬停时使用轻底色，键盘焦点必须有可见描边。

### 内部聊天

- 空会话展示简洁欢迎标题、两个真实可执行的快捷问题和最近问题/能力说明。
- 消息区域占据主要空间，用户与助手消息保持现有语义和错误状态。
- 输入框固定在底部；`Enter` 发送，`Shift+Enter` 换行。
- 输入区不使用虚假的“联网、知识库、文件上传”功能按钮。
- 当前 Provider 为占位模式时，明确展示不可用/降级状态，不伪装成真实 AI 已接通。

## 动效

- 打开：`translateX(18px) + opacity` 进入，时长约 `220ms`，使用平滑减速曲线。
- 关闭：反向收束，时长略短于打开。
- 遮罩独立淡入淡出，不做模糊放大。
- 宽度拖动期间禁用非必要过渡，避免手柄迟滞。
- 形态切换不清空消息，不使用整棵组件重挂载制造假过渡。
- `prefers-reduced-motion: reduce` 下取消位移和缩放，仅保留即时显隐或极短淡变。

## 状态与组件边界

### `AssistantExperienceProvider`

在现有会话能力上增加展示状态：

```ts
type AssistantSurface = "closed" | "quick" | "dock";
```

Provider 负责：

- 当前展示形态与打开来源。
- 快速助手、右侧工作区和全页工作区共享的 `AssistantSession`。
- `openQuickFrom`、`openDockFrom`、`collapseToQuick`、`close`。
- composer 注册/聚焦与触发器焦点恢复。

`/assistant` 是否显示全页工作区由路由决定，不把路由复制进展示状态。

### 新组件

- `AssistantDock`：遮罩、右侧容器、头部操作、焦点边界和关闭行为。
- `useAssistantDockSize`：指针拖拽、键盘调整、视口钳制与本地持久化。
- 可复用的消息列表与输入区：被右侧工作区和全页工作区复用，避免两套聊天行为漂移。

快速助手继续保持轻量，不强行复用全页欢迎布局。

## 可访问性

- 右侧工作区通过 `createPortal(..., document.body)` 挂载，使用对话框语义并提供明确名称。
- 打开后焦点进入输入框；关闭后返回最后触发器。
- 站点壳的背景根节点使用稳定的 `data-assistant-background-root` 标记；工作区打开时只对该背景根设置 `inert` 和 `aria-hidden`，不得对 `body` 或 Portal 容器设置，避免把 Dock 自己一起隔离。
- 打开时保存 `body.style.overflow` 原值并锁定页面滚动；关闭、路由切换、组件卸载和异常清理时恢复原值、移除 `inert`/`aria-hidden`。重复打开关闭不得覆盖最初值。
- 遮罩可点击关闭，但鼠标在面板内部按下后移到遮罩松开不得误关。
- `Esc` 关闭；Tab 焦点不得逃出工作区。
- 拖拽手柄支持键盘并暴露数值语义。
- 状态变化、发送中和错误提示使用现有可访问播报机制。
- 颜色对比、44px 点击面积和减少动效与现有设计系统一致。

## 测试与验收

### 单元/组件测试

1. `quick → dock → quick/closed` 状态切换正确。
2. 快速助手、右侧工作区和 `/assistant` 切换后消息与草稿不丢失。
3. 发送中切换 quick/dock/全页时请求继续、只提交一次，并写回同一会话。
4. 拖拽宽度钳制在 `380–760px` 与视口上限内，并覆盖 `pointercancel`、capture 丢失和卸载清理。
5. 键盘可调整宽度，ARIA 数值同步更新。
6. 合法宽度被保存和恢复；非法值、存储异常和视口跨断点时安全降级且不误覆盖偏好值。
7. 点击遮罩、按 `Esc` 和关闭按钮均关闭，焦点返回触发器；背景 inert 与 body 滚动均被恢复。
8. `/assistant` 只挂载全页工作区，返回门户时不会同时恢复 quick/dock。
9. 移动端全屏且不显示调整手柄；模拟可视视口缩小时输入区可见、消息区可滚动。
10. reduced-motion 下不执行位移动画。

### E2E

- 修正现有 E2E 中已失效的 `.assistant-panel`、`.assistant-message--assistant` 和 `data-motion-state` 选择器，改为稳定的角色、名称或 `data-testid`。
- 验证从门户打开右侧工作区、拖动宽度、刷新恢复宽度、进入 `/assistant` 和返回后的会话连续性。
- 验证发送中切换形态不会重复请求或丢失响应；窗口跨越 `720px` 时手柄、尺寸和滚动锁状态正确。
- 验证桌面与移动端无横向溢出，控制台无错误、页面异常和失败请求。

## 完成标准

- 快速助手、右侧工作区和全页工作区三个入口均可发现、可切换。
- 桌面宽度拖拽和键盘调整稳定，刷新后恢复。
- 移动端为可用的全屏助手，不暴露无效手柄。
- 切换形态不丢消息、草稿或服务状态。
- 占位 Provider 继续诚实降级，服务端合同不变。
- 相关单元测试、E2E、类型、Lint、格式和生产构建全部通过。
