# AI Agent Platform 一期设计说明

## 已确认目标

在当前目录建设前后端全站项目，自建 PostgreSQL 数据库，最终通过 Docker 部署到公司服务器。License、下载、OpenLab 及其他外部资源依赖功能在一期只做页面、类型、接口契约、Mock 和功能开关。

真实品牌和业务资产不作为开工条件。Logo、产品截图、架构图、客户案例、公司信息及正式内容统一先使用明确占位，正式上线前集中替换。

## 推荐方案

采用 Next.js 模块化单体：门户、Nextra 文档、控制台、CMS 和 API 位于一个应用中；数据库、UI 和外部适配器以包边界隔离。部署采用 Docker Compose 的 `proxy + web + db + backup` 结构。

## 设计原则

- 页面不是空白占位，而是完整展示未开放、加载、错误和无权限状态。
- 外部模块不提前发明业务规则。
- PostgreSQL、上传文件、备份均使用独立持久化位置。
- UI 先通过三方向设计稿确定视觉系统，再进入页面实现。
- 品牌素材缺失时只使用显式占位，不伪造 Logo、截图或客户数据。

## 当前约束

- 主色范围：蓝、靛、紫。
- 一期中文优先，结构保留国际化能力。
- 未提供 SMTP 时采用管理员创建账号，不开放自助注册和找回密码。
- 一期以单机 Docker Compose 为部署基线。

## 需要用户确认

- 三个 UI 方向中选择一个或指定混合方案。
- 提供公司服务器环境信息。
- 正式上线前提供 Logo、产品截图、字体和品牌规范；开发期间继续使用占位。

## 自检

- 范围与 PRD 一致。
- 外部模块均明确为占位，不承诺真实业务能力。
- 没有引入微服务、Kubernetes 或独立搜索服务等一期非必要设施。
