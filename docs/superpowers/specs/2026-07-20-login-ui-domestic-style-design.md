# 登录页国内企业风格 UI 设计

## 目标

优化 `/login` 与 `/staff/login` 的视觉和信息结构，使其呈现国内企业平台常见的蓝白双栏登录体验。保留当前账号密码登录、认证安全边界和数据库行为不变；尚未实现的登录方式仅作为明确禁用的入口展示。

## 范围

本次包含：

- 为普通登录和员工登录提供共用的蓝白双栏页面骨架。
- 根据客户与员工身份展示不同的左侧说明、登录方式和辅助入口。
- 为未开放的扫码、手机号、第三方登录、动态口令和企业 SSO 提供禁用占位。
- 优化输入框、按钮、错误提示、跨登录入口和移动端布局。
- 更新与登录页 UI、可访问性和认证契约直接相关的测试。

本次不包含：

- 新增手机号验证码、扫码、第三方登录、动态口令或企业 SSO 的真实认证能力。
- 修改数据库 Schema、迁移、种子数据或既有数据。
- 修改认证 Server Action、Session、权限、限流、审计或重定向规则。
- 改造注册、二次验证、修改密码和重新认证页面。
- 引入远程图片、第三方品牌素材或新的运行时依赖。

## 现状与隔离边界

普通登录和员工登录当前都通过 `AuthPage` 渲染，而 `AuthPage` 还被注册、二次验证、修改密码和重新认证页面使用。直接重做 `AuthPage` 或共享 `AuthShell` 会扩大影响范围，因此本次新增登录专用页面组件，只替换两条登录路由的展示层。

现有表单提交契约保持不变：

| 路由 | 真实字段 | Server Action | 保留行为 |
| --- | --- | --- | --- |
| `/login` | `email`、`password`、可选 `returnTo` | `customerLoginAction` | 客户登录、通用错误提示、注册入口 |
| `/staff/login` | `identifier`、`password`、可选 `returnTo` | `staffLoginAction` | 员工登录、通用错误提示、无公开注册 |

## 视觉系统

采用已确认的“轻蓝双栏式”方向：

- 背景以浅蓝、灰白为主，主操作使用平台现有蓝色变量。
- 桌面端左侧为品牌与辅助信息，右侧为真实账号密码表单。
- 控件采用小圆角、清晰边框和克制阴影，不使用大面积渐变或悬浮炫光。
- 中文为主要信息语言，必要的产品名称保持现有品牌表达。
- 所有占位入口降低视觉权重，并明确显示“即将开放”或“待开放”。
- 二维码区域使用不可扫描的占位图标，不生成容易误导用户的假二维码。

## 页面差异

### 普通登录 `/login`

- 左侧显示平台品牌、客户控制台说明、禁用的扫码入口和第三方登录入口。
- 右侧标题为“登录客户控制台”，默认激活“账号登录”。
- “手机号”和“扫码”作为禁用页签展示。
- 保留客户注册入口，并增加前往员工登录的辅助链接。

### 员工登录 `/staff/login`

- 左侧显示运营工作台品牌、安全访问边界、分域访问和风险控制说明。
- 不展示客户侧二维码或第三方登录品牌。
- 右侧标题为“登录运营后台”，默认激活“账号登录”。
- “动态口令”和“企业 SSO”作为禁用页签展示。
- 保留“员工账号由企业管理员统一创建”的说明，并增加返回客户登录的辅助链接。

## 组件结构

新增登录专用展示组件 `LoginPage`，接收 `customer` 或 `staff` 变体、标题、简介和真实表单内容。组件内部负责：

- 共用双栏结构和响应式布局。
- 根据变体渲染左侧内容与禁用登录方式。
- 提供页面间的辅助跳转。
- 为禁用入口设置原生 `disabled` 与可访问名称。

两条路由继续分别组合 `CustomerLoginForm` 与 `StaffLoginForm`。表单仍由现有 `useActionState`、`useFormStatus` 和 Server Action 驱动。

登录页新增独立的 `login-page.css`，所有规则都限定在登录专用页面根类名内。现有 `login-form.css` 还被注册和员工安全流程复用，因此不进行会影响其他页面的全局重写；登录页所需的表单视觉覆盖必须限制在登录专用根节点内。

## 交互与状态

- 真实输入字段保留可见标签，并增加中文占位提示。
- 提交按钮继续反映 `pending` 状态并禁止重复提交。
- 认证失败继续只显示稳定、通用的错误信息，不暴露内部错误。
- 未开放入口不响应点击、不提交表单、不触发请求。
- `returnTo` 隐藏字段原样保留，登录成功后的现有重定向行为不变。
- 键盘焦点使用清晰的蓝色轮廓，禁用入口具有可辨识的禁用状态。

## 响应式与可访问性

- 宽屏使用左右双栏，表单区域保持合理最大宽度。
- 平板逐步缩窄左栏；手机切换为单列，隐藏二维码和较长安全说明，仅保留品牌摘要与真实表单。
- 输入框、按钮和可点击链接的操作高度不低于 44px。
- 保留语义化 `main`、标题层级、表单标签、`aria-live` 错误提示和原生禁用属性。
- 遵守 `prefers-reduced-motion`，不依赖动画表达状态。

## 验证

实施完成后至少执行：

- 更新并运行 `AuthPage`/登录专用组件、普通登录表单、员工登录表单和两条登录路由的组件测试。
- 验证表单字段名、Server Action 引用、`returnTo`、pending 状态和稳定错误文案未改变。
- 验证客户与员工占位入口分别存在、均为禁用状态，且两页互相链接正确。
- 验证所有交互目标满足 44px 下限，窄屏样式不产生横向溢出。
- 运行 Web 包的相关 Vitest、TypeScript 类型检查和 ESLint。
- 实际启动页面，检查桌面端与移动端布局、焦点状态和浏览器控制台。

## 验收标准

- `/login` 与 `/staff/login` 呈现已确认的轻蓝双栏企业登录风格。
- 两页共享一致的视觉系统，同时明确区分客户与员工入口。
- 账号密码登录、错误处理和成功重定向继续按现有逻辑工作。
- 所有未实现能力均明确禁用，且不会发起网络请求。
- 数据库与认证后端没有任何代码或迁移变更。
- 注册、二次验证、修改密码和重新认证页面不受本次样式改造影响。
