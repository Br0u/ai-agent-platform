# 首页内容密度与滚动入场设计

> 日期：2026-07-16  
> 状态：已确认  
> 参考：首页当前实现与阿里云首页公开动效样式

## 1. 背景

首页 Hero 之后的内容区使用 1360px 宽容器、大标题、约 690–720px 的双栏面板和 148–180px 的列表行。单个组件占用面积过大，页面横向铺得过满，纵向阅读节奏也过松。

本次只调整第 2–5 个内容区：平台能力、企业能力、解决方案、资源入口。Hero 与底部深色私有化 CTA 保持现状。

## 2. 目标

1. 将第 2–5 区的内容宽度从 1360px 收敛到约 1120px。
2. 将面板、卡片和列表行高度整体压缩约 20–30%。
3. 同步缩小后四区的标题、图标、圆角、间距和内边距，避免只缩容器导致内部仍显臃肿。
4. 参考阿里云首页，改为区块进入视口后再播放上移淡入，文字和列表分层出现。
5. 保持现有内容、链接、语义结构和移动端可读性。

## 3. 非目标

- 不修改 Hero 的 DOM、宽度、字号、截图、按钮、背景和现有入场效果。
- 不修改底部深色 CTA 的 DOM、高度、布局和动效。
- 不修改首页文案、数据数组、链接目标或区域顺序。
- 不引入通用动画库、滚动劫持、视差、缩放弹跳或循环动画。
- 不重构首页以外的页面和共享导航。

## 4. 已确认方向

采用“中度收窄”方案：保持当前玻璃卡片和左右双栏语言，但从展示型大画布调整为更适合连续阅读的内容宽度。

| 项目 | 当前 | 调整目标 |
| --- | --- | --- |
| 后四区内容宽度 | 最大 1360px | 最大约 1120px |
| 区域纵向留白 | 72–128px | 64–96px |
| 双栏介绍面板 | 690–720px | 约 500–560px |
| 平台列表行 | 180px | 约 128–140px |
| 企业列表行 | 154px | 约 112–124px |
| 方案列表行 | 148px | 约 112–124px |
| 资源列表行 | 172px | 约 116–128px |
| 后四区主标题 | 46–78px | 约 38–56px |
| 主图标 | 64px | 48–52px |

数值允许在浏览器验证时小幅修正，但不得回到当前的宽松密度。

## 5. 布局设计

### 5.1 作用域

在 `.home` 下增加仅供第 2–5 区使用的窄容器规则，不修改全局 `.home-frame`，从而保证 Hero 与底部 CTA 的宽度不变。

四个目标区域继续使用现有 `data-home-region`：

- `platform`
- `enterprise`
- `solutions`
- `resources`

### 5.2 平台能力区

- 四张能力卡继续保持四列，但降低卡片高度、图标尺寸和内部留白。
- 下方平台流程继续保持左侧介绍、右侧四行能力的双栏结构。
- 左侧插图缩小，不再依赖 720px 高度撑开面板。
- 右侧标题与说明保留同一行，但缩小列间距和行高。

### 5.3 企业能力区

- 标题面板减高，标题字号与上下留白同步收敛。
- 四条企业能力压缩为更紧凑的横向信息行。
- 描述仍保留，不改成只显示标题的卡片。

### 5.4 解决方案与资源区

- 两区继续使用左侧介绍、右侧列表，避免无关结构重写。
- 左侧插图缩小并限制占位高度。
- 右侧列表行、图标、编号、箭头和间距同步减小。
- “视觉检索解决方案”的子能力标签和强调边保持不变。

### 5.5 响应式

- `1180px` 以下继续切为单列，不新增断点体系。
- 平板单列时取消桌面固定高度，插图限制在较小宽度内。
- 移动端保留当前信息层级，但进一步压缩面板内边距和列表行最小高度。
- 不允许产生横向滚动或为了紧凑而截断正文。

## 6. 动效设计

### 6.1 参考与取舍

阿里云首页公开样式使用进入视口触发，并提供三类效果：`fadein`、`slideup`、`zoomin`。其中淡入从约 20px 模糊恢复，滑入从约 48px 下方进入，基础时长约 0.3s，并通过 0.1–0.8s 延迟形成层级。

本页保留“进入视口触发”和“分层延迟”，但降低刺激强度：

- 区块位移：32px → 0。
- 区块时长：约 520ms，使用现有缓出曲线。
- 标题文字：10px 模糊、10px 位移、约 480ms。
- 列表项：18px 位移、约 440ms，相邻项延迟 60ms。
- 每个区域只播放一次，进入后取消观察。
- 不使用缩放和回弹，避免卡片显得更膨胀。

### 6.2 触发边界

新增一个轻量客户端观察器，只观察第 2–5 区。区域接近视口底部约 96px 时触发；Hero 与底部 CTA 不进入该观察器。

服务端仍输出完整可读内容。客户端脚本不可用时，内容必须保持可见，不得因初始隐藏永久消失。

### 6.3 分层顺序

每个区域按以下顺序出现：

1. kicker 与主标题；
2. 简介和按钮；
3. 能力卡或主面板；
4. 列表行按顺序递进；
5. 装饰插图最后淡入。

### 6.4 减少动态效果

`prefers-reduced-motion: reduce` 下取消位移、模糊、延迟与过渡，所有内容立即显示。现有 Hero 与底部 CTA 的降级规则继续生效。

## 7. 实现边界

- `apps/web/src/components/home.css`：后四区密度、响应式与入场样式。
- `apps/web/src/components/home-sections.tsx`：只增加必要的动效标记，不修改文案和结构含义。
- 新增一个小型客户端组件负责 `IntersectionObserver`，不把整套首页内容改为客户端渲染。
- `apps/web/src/app/page.tsx`：只挂载观察器或动效根组件。

观察器只负责添加可见状态类；所有视觉参数留在 CSS。组件卸载时断开观察器，已经显示的区域不重复播放。

## 8. 验证

### 8.1 自动验证

- 保持现有首页区域顺序与内容测试通过。
- 增加测试，确认只标记 `platform`、`enterprise`、`solutions`、`resources` 为滚动入场目标。
- 模拟 `IntersectionObserver`，确认进入后添加可见状态并取消观察。
- 运行 Web 单测、lint、类型检查。

### 8.2 浏览器验证

在桌面宽屏、平板和移动端检查：

- Hero 与底部 CTA 的截图对比无布局变化。
- 第 2–5 区最大宽度约 1120px，页面无横向滚动。
- 后四区明显减高，不出现文字拥挤、重叠或截断。
- 动画只在滚动进入时播放一次，标题和列表有清楚但克制的层级。
- 开启减少动态效果后，内容立即可见且布局不跳动。
