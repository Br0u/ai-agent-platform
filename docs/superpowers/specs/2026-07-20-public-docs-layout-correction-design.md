# 公开文档布局修正设计

## 背景

公开 `/docs` 已切换为 CMS 发布数据，但桌面端把长英文导航代码放进过窄网格列，导致代码与中文标题重叠；总览页仍保留详情页的空右栏，搜索条也没有与阅读网格对齐。

## 已确认方向

- 采用局部修正，不重做现有品牌视觉。
- 公开文档界面不显示 `D0`、`QUICK_START`、slug、`OVERVIEW` 等英文代码；CMS 管理界面的导航代码字段保持不变。
- 侧栏、移动目录、总览卡片和搜索结果只显示中文标题、摘要及必要状态。
- 总览页使用“左侧导航 + 弹性正文”两列网格，不再保留空的页内目录列；详情页继续保留真正有用途的页内目录列。
- 搜索条收进与文档内容相同的最大宽度，并减少顶部空白。

## 响应式边界

- 宽屏：总览正文获得更合理宽度，侧栏与正文保持清晰间距。
- 中屏：隐藏详情页页内目录，总览与详情都保持两列。
- 小屏：继续使用现有折叠目录；去掉代码后目录项为单列中文标签，点击区域不缩小。

## 验证

- 组件测试证明公开导航、卡片和搜索结果不渲染导航代码，同时仍保留正确链接和搜索能力。
- `/docs` 与详情页测试保持通过。
- 在真实浏览器检查桌面和移动视口：无重叠、无横向滚动、搜索条与内容网格对齐。

## 版本控制边界

本次设计、计划和实现保持未暂存；只有用户再次明确授权后才提交。
