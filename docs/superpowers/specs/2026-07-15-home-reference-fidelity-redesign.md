# 首页参考图高还原重设计规格

## 1. 目标

将用户提供的五张参考图按顺序实现为主页的五个连续区域，在真实 HTML、可访问交互和完整响应式的前提下，高保真还原参考图的构图、比例、材质、字体层级、图标与蓝紫 3D 插画。

目标不是把参考图直接切片成背景图，而是使用独立组件、真实文字、真实链接和本地静态素材完成高还原实现。

## 2. 已确认范围

主页顺序固定为：

1. 现有 Navbar。
2. Hero 首屏。
3. 能力链与平台全流程。
4. 企业边界。
5. 行业场景。
6. 资源入口。
7. 现有深色私有化部署收口。
8. 现有 Footer。

Navbar 不做任何代码或样式修改。现有 Footer 和深色私有化部署收口保留；深色收口只允许调整与上一浅色区域之间的视觉衔接，不改变内容和链接。

文案与信息结构以参考图为准，现有 CTA 和资源入口的路由地址保持不变。主页必须同时支持桌面、平板和移动端。交互采用克制动效，并支持 `prefers-reduced-motion`。

## 3. 视觉系统

### 3.1 整体氛围

- 页面使用接近 `#F4F7FF` 的冷白蓝底色。
- 以低饱和蓝、靛、紫径向光晕制造参考图中的空气感，不使用高饱和大面积紫色。
- 内容表面使用 64%–88% 透明白色、低对比浅边框、柔和蓝靛阴影和受控的背景模糊。
- 大区块圆角为 32–48px，列表和小卡圆角为 18–26px，胶囊标签和按钮使用全圆角。
- 标题使用深海军蓝；`AI`、`行业场景`、`开始` 等参考图重点词使用蓝紫渐变。
- 阴影只表达层级，不使用厚重黑色投影。

### 3.2 字体与排版

- 中文标题保持紧凑、粗重和高对比，桌面主标题使用大尺度 `clamp()`，移动端按视口缩放。
- 英文分区标签使用大写等宽字体和加宽字距。
- 正文使用灰蓝色，行高约 1.7–1.85，并保证 WCAG AA 对比度。
- 中文标题不得出现孤立单字行；移动端通过字号、最大宽度和手动断行共同控制。

### 3.3 图标与插画

- 功能图标使用统一的白色线性符号、蓝紫渐变内芯、半透明外壳和柔和外发光。
- 生成三组与参考图一致的本地透明静态素材：平台环、行业场景平台、资源文件夹。
- 生成素材应以高分辨率透明 PNG 为母版，并提供适合网页加载的 WebP 版本。
- 装饰插画不承载信息，使用空替代文本；产品截图保留明确的替代文本和来源说明。
- 不依赖运行时远程图片地址，不伪造新的产品操作界面。

## 5. 响应式规则

- 大于等于 1180px 时使用参考图的桌面左右构图和大尺度留白。
- 760–1179px 时左右分区允许收窄或转为单列，能力卡保持两列。
- 小于 760px 时所有左右分区改为单列：叙事面板在上，列表面板在下。
- Hero 移动端先显示文案，再显示产品截图。
- 3D 插画缩小并锚定在叙事面板底部，不遮挡标题、正文和按钮。
- 产品截图保持原始比例，所有面板不得产生横向溢出。
- 大标题、面板间距、内边距和圆角通过 `clamp()` 与媒体查询缩放。

## 6. 交互与动效

- 首屏与后续分区进入使用 `opacity` 与 `translateY(8px)`，时长 280–420ms。
- 可点击列表行 hover 时上浮 2–3px、边框提亮、阴影略增强，箭头右移 4px。
- 按钮按下使用 `scale(0.98)`。
- 3D 插画不持续旋转；只允许背景光晕极慢漂移。
- 不使用 `transition: all`，只动画 `transform`、`opacity`、颜色、边框色和阴影。
- `prefers-reduced-motion: reduce` 下取消位移、漂移和错峰动画。
- 所有交互目标至少 44×44px，`:focus-visible` 轮廓在浅色和深色背景上都必须清晰。

## 7. 组件与数据边界

主要改动限定在：

- `apps/web/src/components/home-sections.tsx`
- `apps/web/src/components/home-content.ts`
- `apps/web/src/components/home.css`
- `apps/web/src/app/page.test.tsx`
- 首页专用静态素材目录

保留现有服务端组件模式，不引入首页客户端状态、CMS 请求或远程运行时依赖。静态内容继续由 `home-content.ts` 提供，组件只负责渲染。

组件边界保持为 Hero、能力链、平台全流程、企业边界、行业场景、资源入口和现有深色收口。能力链与平台全流程可以在页面顺序上连续，但组件内部职责仍分离。

Navbar、共享导航样式和导航测试不在改动范围内。

## 8. 素材加载与异常状态

- 所有静态图片使用明确宽高或固定 `aspect-ratio`，防止加载时布局跳动。
- 产品截图加载失败时保留玻璃容器比例和可理解的文字说明。
- 装饰插画缺失不得导致标题、正文、按钮或列表布局塌陷。
- 静态导入失败应在构建阶段暴露，不做静默远程回退。
- 不支持 `backdrop-filter` 的浏览器使用不透明度更高的冷白实色回退。

## 9. 验收标准

### 9.1 视觉

- 1440px 及更宽桌面视口下，五个区域的构图、比例、圆角、留白、颜色、阴影和插画位置与参考图保持一致。
- 参考图中的功能图标、副标题、编号、重点渐变词和 S5 高亮均完整出现。
- Navbar 未发生视觉或行为变化。
- 深色私有化收口与 Footer 保留。

### 9.2 响应式与可访问性

- 390px、768px、1440px 视口无横向溢出、文字裁切或图片变形。
- 键盘可访问全部按钮和链接，焦点状态清晰。
- 正文和重要标签满足 WCAG AA 对比度。
- 减少动态效果模式下不存在位移、漂移和错峰动画。

### 9.3 工程质量

- 首页组件测试通过，并覆盖区域顺序、关键文案、链接、图片替代文本和稳定结构选择器。
- TypeScript 类型检查、ESLint、生产构建通过。
- 使用真实浏览器分别检查桌面、平板、手机和减少动态效果模式。
- 保存最终浏览器截图，与五张参考图逐区对比并修正明显偏差。
- 不覆盖工作区内与本次主页改造无关的未提交改动。

## 10. 不在范围内

- Navbar 重设计或导航行为修改。
- Footer 内容重写。
- 深色私有化收口内容与链接修改。
- 新增后端接口、CMS、用户状态或埋点。
- 把参考图直接作为整区背景图。
