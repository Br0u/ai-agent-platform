# 价格计算页与 M 助手设计

## 1. 目标

本次新增两个公开门户能力：

1. 提供“价格计算”页面雏形，帮助潜在客户整理部署和采购需求，但不产生真实价格或正式报价。
2. 提供全站浮动的 M 助手入口，支持预设问题和自由输入，并为未来接入 AI 服务保留稳定接口。

本阶段不连接外部 AI、不执行真实价格计算、不写入数据库，也不采集或持久化客户提问正文。

## 2. 信息架构

### 2.1 价格计算

- 页面路由：`/pricing`
- 导航位置：`支持 → 商务服务 → 价格计算`
- 页脚位置：`支持与商务联系 → 价格计算`
- 页面标题：`价格计算`
- 定位说明：根据部署方式、使用规模、功能模块和服务周期整理需求，当前不产生真实价格。

价格计算属于售前决策工具，不放在产品功能、客户账单或后台管理分类中。

### 2.2 M 助手

M 助手只出现在路由注册表中 `group: "public"` 的内容页面。可见性通过独立纯函数根据路由注册信息判定，未知路由和受保护路由默认不显示。以下公开身份页面额外排除：

- `/login`、`/register`
- `/staff/**`
- `/console/**`
- `/admin/**`

入口固定在页面右下角。点击后展开桌面客服小窗；移动端使用底部抽屉。

## 3. 视觉系统

遵循现有 `docs/design/brand-spec.md` 和 `packages/ui/src/tokens.css`：

- 画布：`#F7F8FB`
- 表面：`#FFFFFF`
- 主操作蓝：`#3A67B1`
- 结构靛蓝：`#4C91EB`
- 智能状态紫：`#9277DC`
- 主文字：`#101838`
- 次级文字：`#566078`
- 分隔线：`#DCE3EF`

蓝紫连续色只用于细窄状态提示或小范围品牌信号，不使用大面积渐变背景。页面保持工业精确、企业可信、产品导向的现有视觉语言。

### 3.1 价格页布局

桌面端使用双栏结构：

- 左侧为需求配置区。
- 右侧为需求摘要和商务行动区。
- 窄屏按配置区、摘要区顺序纵向排列。

配置项第一期包括：

- 部署方式（必选、单选、默认 `local-private`）：本地私有化 `local-private`、专有云 `dedicated-cloud`、待商务确认 `tbd`。
- 使用规模（必选、单选、默认 `pilot`）：体验验证 `pilot`、部门级 `department`、企业级 `enterprise`。
- 功能模块（必选、多选、默认空）：Agent Studio `agent-studio`、Knowledge Base `knowledge-base`、Workflow `workflow`、Model Gateway `model-gateway`、Agent Runtime `agent-runtime`、Observability `observability`。至少选择一个模块后才能提交商务需求。
- 服务周期（必选、单选、默认 `tbd`）：一年 `1y`、三年 `3y`、待商务确认 `tbd`。

配置选择只更新需求摘要，不生成数字金额。摘要区固定显示“在线估算尚未开放，最终价格以商务报价为准”。主行动按钮为“获取正式报价”。

### 3.2 机械折线 M

M 小人采用“机械折线 M”造型：

- M 字形本身构成躯干和腿部。
- 中央凹点形成头部视觉焦点。
- 外侧延伸短机械臂。
- 蓝色主体、靛色结构、紫色智能状态灯。
- 轮廓紧凑，在约 64px 显示尺寸下仍能辨认字母 M。
- 不复制现有 Logo，不包含文字，不使用分离式装饰特效。

素材源文件为透明 PNG，建议画布 `1024 × 1024px`，保存在 `docs/design/assets/m-assistant/source.png`；生产输出为透明 WebP，尺寸 `256 × 256px`，保存在 `apps/web/public/assets/assistant/m-assistant.webp`。CSS 只负责轻微整体呼吸和状态切换；不使用 CSS 图形拼凑替代角色素材。若用户启用减少动态效果，关闭持续动画。

本角色只作为网站入口素材，不生成或安装完整 Codex Pet 动画图集。

## 4. 交互设计

### 4.1 价格页

用户修改配置后，右侧需求摘要即时同步。未选择任何功能模块时，“获取正式报价”不可用并显示原因。点击后跳转 `/contact`，使用 `URLSearchParams` 生成以下查询参数：

- `source=pricing`
- `deployment=<稳定 ID>`
- `scale=<稳定 ID>`
- `modules=<按字母排序、英文逗号分隔的稳定 ID>`
- `term=<稳定 ID>`

联系页只接受上述允许列表中的稳定 ID；非法值忽略，不把完整自由文本放入 URL。

第一期保留 `POST /api/v1/pricing/estimate` 接口契约，但页面不调用该接口。接口返回结构化的 `not_available` 状态，不返回虚构金额。

### 4.2 M 助手

桌面小窗宽度为 `min(360px, calc(100vw - 32px))`，最大高度为 `min(480px, calc(100dvh - 96px))`；移动端底部抽屉最大高度为 `75dvh`。内容包含：

- 标题和“AI 服务尚未接入”状态。
- 简短欢迎语。
- 三个预设问题：
  - `如何开始了解平台？`：引导至快速开始文档。
  - `如何获取部署支持？`：引导至商务咨询。
  - `如何提交产品问题？`：引导至客户支持。
- 消息历史区域。
- 自由输入框和发送按钮。
- 帮助中心与商务咨询兜底入口。

点击预设问题或发送自由输入后，前端调用 `POST /api/v1/assistant/chat`。第一期由 `PlaceholderAssistantProvider` 根据上述三个精确问题返回对应固定引导；其他内容统一返回通用占位回答，并带 `mode: "placeholder"`。未来接入 AI 时替换 Provider，不改变前端协议。

发送前去除首尾空白。空内容不发送；消息按 Unicode 字符计数，最多 500 个字符。请求进行中禁用发送按钮、输入框和预设问题，防止重复提交。失败时保留原输入并显示“重试”按钮；每次点击重试只重新发送一次相同请求。

小窗状态保存在 `SiteShell` 组件内存中，因此同一浏览器文档内的客户端页面导航会保留会话；刷新或关闭页面后清空。服务异常时保留用户输入，并显示帮助中心和商务咨询入口，不伪装成 AI 已成功回答。

## 5. 模块边界

### 5.1 价格模块

- 页面组件：负责布局和表单组合。
- 价格配置：维护选项和展示文案。
- 摘要构建器：纯函数，根据选项生成需求摘要。
- 接口契约：定义未来估算请求、成功结果和未开放状态。

价格模块不依赖数据库、认证或助手模块。

### 5.2 助手模块

- `AssistantLauncher`：浮动入口和未读/状态表现。
- `AssistantPanel`：小窗、消息列表、预设问题和输入。
- `AssistantProvider`：服务端回答接口。
- `PlaceholderAssistantProvider`：第一期固定回答实现。
- API Route：校验请求并转发 Provider。

助手组件由公开门户壳层挂载。路由可见性由明确规则控制，不在每个页面重复判断。

## 6. 接口草案

### 6.1 助手

请求：

```json
{
  "message": "如何获取部署支持？",
  "context": {
    "pathname": "/pricing"
  }
}
```

响应：

```json
{
  "mode": "placeholder",
  "message": "AI 服务尚未接入。你可以先查看帮助中心或联系商务顾问。",
  "suggestedActions": [
    { "label": "帮助中心", "href": "/help" },
    { "label": "商务咨询", "href": "/contact" }
  ]
}
```

接口规则：

- 成功：HTTP `200`，返回上述响应。
- 空内容或超过 500 个 Unicode 字符：HTTP `400`。
- Provider 暂时不可用：HTTP `503`，不回显原消息。
- 不支持的方法由框架返回 HTTP `405`。
- 服务端日志只记录请求 ID、状态码和耗时，不记录消息正文。

校验错误响应：

```json
{
  "mode": "placeholder",
  "error": {
    "code": "invalid_message",
    "message": "请输入 1 至 500 个字符的问题。"
  }
}
```

服务异常响应使用相同结构，错误码为 `assistant_unavailable`，提示用户使用帮助中心或商务咨询。

### 6.2 价格估算

第一期请求：

```json
{
  "deployment": "local-private",
  "scale": "pilot",
  "modules": ["agent-studio"],
  "term": "tbd"
}
```

请求字段使用第 3.1 节稳定 ID，`modules` 至少包含一个无重复值。第一期所有合法请求返回 HTTP `501`：

```json
{
  "status": "not_available",
  "message": "在线估算尚未开放，最终价格以商务报价为准。"
}
```

无效请求返回 HTTP `400`：

```json
{
  "status": "error",
  "error": {
    "code": "invalid_configuration",
    "message": "价格配置无效。"
  }
}
```

## 7. 可访问性与响应式

- 浮动入口和发送按钮触控区域不小于 44px。
- 小窗使用对话框语义，打开后移动焦点，Esc 关闭，关闭后焦点返回入口。
- 消息区域使用合适的动态播报，不重复朗读全部历史。
- 移动端采用底部抽屉并避开安全区。
- 支持 `prefers-reduced-motion`。
- 价格表单具备可见标签、键盘操作和明确状态说明。

## 8. 验收标准

- `/pricing` 已注册并可从支持菜单与页脚访问。
- 配置选择能更新需求摘要，但任何状态下都不显示虚构价格。
- 商务跳转携带安全的配置标识。
- M 助手只显示在允许的公开门户页面。
- 助手支持打开、关闭、Esc、焦点回收、预设问题和自由输入。
- 占位 Provider 返回稳定协议并明确标注 AI 尚未接入。
- 不调用外部 AI，不写数据库，不记录提问正文。
- 路由、导航、摘要构建器、可见性规则、接口校验和聊天状态具备自动化测试。
- 桌面和移动端真实浏览器验收无横向溢出、无控制台错误，减少动态效果生效。

## 9. 非目标

- 真实价格模型、优惠、税费、币种和正式报价单。
- 登录后保存方案或关联客户组织。
- AI 模型、知识库、流式输出、会话持久化和人工客服转接。
- 完整 Codex Pet 图集或 Codex 本地宠物安装。
- CMS 管理价格配置、助手提示词或问答内容。
