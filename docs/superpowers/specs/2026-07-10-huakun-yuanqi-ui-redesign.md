# 华鲲元启门户 UI 重设计规格

## 1. 目标与范围

本次重设计把门户从泛化的“AI Agent Platform”模板，调整为有明确产品身份的“华鲲元启 AI开发赋能平台（TGDataXAI）”门户。

首页第一优先服务企业决策者与潜在客户，同时为IT架构师和开发者保留清晰的文档、版本、兼容与支持入口。

本次实施范围：

- 全局品牌名称、页头、页脚和导航。
- 首页完整视觉与信息层级。
- 通用页面壳、外部功能占位页的视觉适配。
- 桌面和移动响应式行为。

不在本次实施中扩展登录、CMS、业务数据或外部集成功能。

## 2. 产品层级

门户只建立以下主从关系：

1. 华鲲元启是主产品与门户主品牌。
2. TGDataXAI作为产品英文标识和技术标识出现。
3. 模型仓库、知识工程、流程编排、训练中心、推理中心、评估中心是平台核心能力。
4. 视觉检索属于基于华鲲元启构建的多模态行业解决方案，不作为平级平台或首页主品牌。
5. “融多多”和TGHCI只在解释全栈方案或算力底座时出现，不抢占华鲲元启主叙事。

## 3. 设计方向

采用“工业编辑式结构 + 产品控制台证据”的组合方向。

### 3.1 视觉原则

- 使用严格网格、横向分隔线、大尺度中文标题和真实产品界面证据。
- 页面以白色和浅灰为主，深靛承担结构，蓝色承担操作，紫色只表示模型或智能状态。
- 蓝紫连续色只允许出现在细窄品牌光谱、状态条或真实产品截图中，不使用大面积渐变背景。
- 不使用渐变球、漂浮装饰、三栏功能卡、圆角卡片堆叠和无意义统计数字。
- 企业价值通过真实能力、部署方式和产品结构表达，不使用虚构客户、评价或数据。

### 3.2 设计系统

- 字体尺度：80 / 56 / 40 / 28 / 18 / 15 / 13；移动端主标题降为48。
- 中文字体：优先使用清晰、收紧的中文黑体栈；英文技术标签使用等宽字体。
- 页面网格：桌面12列，最大内容宽度1440px，左右留白80px；移动端20px。
- 间距：8px基础网格，主要段落使用24 / 40 / 64 / 96 / 144。
- 圆角：普通内容区不使用圆角；按钮和小型状态控件最多4px。
- 动效：160ms交互反馈、280ms内容显现，使用统一缓动；不做持续漂浮动画。
- 交互目标：所有可点击区域至少44×44px，键盘焦点清晰可见。

### 3.3 色彩

- 主蓝：`#3A67B1`
- 信号青：`#56C0F8`
- 结构靛：`#4C91EB`
- 智能紫：`#9277DC`
- 母品牌红：`#D91F26`
- 深色文字：`#101838`
- 次级文字：`#566078`
- 页面底色：`#F7F8FB`
- 内容表面：`#FFFFFF`

红色只用于真实华鲲振宇标识或风险状态，不与蓝紫主交互色竞争。

## 4. 全局框架

### 4.1 页头

- 左侧为“华鲲元启”文字标识，下一行或旁侧显示`TGDataXAI`。
- 桌面导航：平台能力、行业方案、文档、版本与兼容、支持。
- 右侧保留登录入口。
- 页头底部增加2px品牌光谱线，作为全站唯一连续色装饰。
- 移动端使用原生`details/summary`折叠菜单，避免为了导航引入客户端JavaScript。

### 4.2 页脚

- 显示产品全称、关键入口和公司信息占位。
- 不复制营销型多列链接矩阵；保持两层信息和清晰分隔。

### 4.3 通用页面壳

- 普通搭建页使用编号、页面标题、状态说明和相关入口，不使用大圆角空状态卡。
- 外部功能占位页明确显示“功能尚未开放”和稳定错误码，不出现可点击的假操作按钮。

## 5. 首页信息架构

### 5.1 Hero：先讲企业价值，再给产品证据

- 技术标签：`国产算力 · 私有化部署 · 企业级AI开发`
- 主标题：`让企业 AI 从模型走向业务`
- 产品名：`华鲲元启 AI开发赋能平台`
- 支撑文案：说明异构算力调度、低代码开发、模型全生命周期和私有化落地。
- 主操作：`了解平台`，链接`/product`。
- 次操作：`阅读文档`，链接`/docs`。
- 右侧展示来自平台彩页的真实界面截图，附小字“界面示意 · 来源于产品彩页”；不伪造新的产品UI。

### 5.2 企业价值条

用一条无卡片的横向信息带表达四项能力：

- 私有化部署
- 异构算力调度
- 低代码智能体开发
- 模型全生命周期管理

这些是能力标签，不包装为统计数字。

### 5.3 平台能力架构

标题：`一套平台，贯通企业 AI 开发全流程`

采用纵向流程与横向模块相结合的结构：

1. 数据与知识：知识库、多模态文档、知识图谱、数据源。
2. 开发与编排：流程编排、Prompt、MCP、智能体发布。
3. 模型与运行：模型仓库、训练、推理、评估、部署。
4. 企业底座：权限管理、用户管理、算力分配。

不照搬彩页架构图，不声明彩页以外的真实运行链路。

### 5.4 企业优势

使用四条全宽编辑式陈述，每条包含编号、短标题和一段说明：

1. 数据留在企业边界内。
2. 复杂文档和非结构化数据可进入知识工程。
3. 低代码编排缩短应用落地路径。
4. 权限、数据和算力统一管控。

### 5.5 行业解决方案

采用可扫描的文本列表，不做行业Logo墙或客户案例卡片。

- 知识问答与知识加工
- 数据问答与报告生成
- 知识图谱
- 图像与多模态处理
- 视觉检索解决方案

视觉检索行必须带`基于华鲲元启的行业子能力`说明，详情内容只引用彩页已确认的即时检索、持续布控、自然语言配置和预警管理。

### 5.6 高频入口

用紧凑的链接表格提供文档、版本、兼容矩阵和支持入口。每行包含入口名称、用途和箭头，不使用四张功能卡。

### 5.7 收口

最终区块强调私有化部署与商务沟通：

- 主操作：`联系商务`
- 次操作：`查看部署文档`
- OpenLab仅显示未开放状态，不作为主CTA。

## 6. 素材策略

- 实施阶段从用户提供的彩页中提取华鲲元启文字标识和一张平台首页截图，作为当前原型素材。
- 彩页截图最大显示宽度限制在保证清晰度的范围内，并保留来源说明。
- 视觉检索设备图和场景图不进入首页Hero，只在后续方案详情使用。
- 正式上线前必须替换为独立Logo、原始平台截图和获得授权的设备图。
- 缺失素材使用明确占位，不绘制假Logo、假产品界面或CSS设备轮廓。

## 7. 组件边界

- `AppShell`：品牌、桌面/移动导航、页脚。
- `HeroEvidence`：价值主张、操作和真实截图证据。
- `CapabilityRail`：四项企业能力横向信息带。
- `PlatformFlow`：四层平台结构。
- `EnterpriseProof`：编号式企业优势。
- `SolutionIndex`：行业方案列表，负责表达视觉检索的从属关系。
- `ResourceTable`：高频入口表格。
- `FeaturePlaceholderPage`：普通搭建页与外部功能占位页。

组件只接收展示内容，不在本次改造中引入CMS请求或客户端状态。

## 8. 数据流与异常状态

- 首页内容在当前阶段由服务端组件中的静态、已核验文案提供。
- 所有链接使用已有路由注册表，不新增不存在的操作路径。
- 产品截图加载失败时显示固定比例的资产占位和文字说明，页面布局不得塌陷。
- 外部功能继续通过现有Provider状态返回`FEATURE_DISABLED`，UI显示未开放状态。
- 不展示未核验的版本号、客户名称、性能指标或认证结论。

## 9. 验收标准

### 9.1 内容

- 全站主品牌统一为“华鲲元启”，TGDataXAI作为副标识。
- 首页30秒内可理解平台定位、企业价值和文档入口。
- 视觉检索只作为行业解决方案/子能力出现。
- 页面无虚构指标、客户、评价和产品界面。

### 9.2 视觉与交互

- 1440px桌面和390px移动视口无横向溢出或内容裁切。
- 导航、主操作和移动菜单可通过键盘使用。
- 所有点击目标不小于44px。
- 真实产品截图有来源说明和替代文本。
- 浏览器控制台无404、JavaScript错误和React警告。

### 9.3 工程质量

- 首页与AppShell先更新失败测试，再实施代码。
- 单元测试、类型检查、ESLint、格式检查和生产构建全部通过。
- 使用真实浏览器检查首页、占位页及移动菜单主路径。
- 不增加非必要客户端JavaScript和外部运行时资源。
