# 企业门户全站菜单设计

> 日期：2026-07-10  
> 状态：待用户书面审查  
> 设计方向：方案 A——精简顶栏 + Mega Menu  
> 范围：公开门户、客户控制台、CMS 运营后台、移动端与页脚导航

## 目标

1. 覆盖 `AI Agent Platform 企业级产品门户 PRD V2.0.docx` 中的完整信息架构，不再出现“路由存在但菜单不可达”。
2. 保持公开门户顶栏清晰，通过分组下拉容纳完整目录。
3. 为 Console 和 CMS 建立独立、可扩展的侧边栏菜单。
4. 下载、OpenLab、License 等外部能力保留入口和占位状态，不伪造可用能力或业务数据。
5. 桌面、平板、手机共用同一份菜单配置，避免入口遗漏和命名漂移。

## 设计原则

- 首页由品牌标识返回，不单独占用顶栏位置。
- 一级入口保持稳定，二三级目录通过 Mega Menu、侧边栏或移动端折叠组展示。
- 菜单项的名称、地址、状态和权限来自统一配置，不在不同组件中重复手写。
- 未开放模块允许进入占位页面，并明确显示“尚未开放”；不使用假下载、假申请或假授权结果。
- 当前页面、悬停、键盘焦点、禁用和未开放状态必须可区分。
- 没有正式图标时以文字导航为主，不制造临时品牌图形。

## 视觉系统

- 延续现有蓝、靛、紫色系和顶栏细窄光谱线，不使用大面积渐变。
- 英文书法字体只用于品牌名；菜单使用现有正文/展示字体，保证中英文可读性。
- 公开门户顶栏维持约 76px；品牌在左、一级导航居中、账号操作在右。
- Mega Menu 宽约 1080–1200px，使用 3–4 列文字导航和细分隔线，不堆叠圆角卡片。
- 当前一级栏目使用靛蓝下划线；当前子页面使用浅蓝底和深色文字。
- 未开放状态使用小型文字标签，不降低菜单项可读性。
- 所有点击目标不小于 44×44px。

## 交互

- 鼠标悬停和点击都能打开下拉菜单。
- 离开触发器和面板后短暂延迟关闭，避免鼠标跨越间隙时误关。
- 同一时间只展开一个菜单；点击外部或按 Esc 关闭。
- 支持 Tab、Enter、Esc 和方向键导航。
- 键盘焦点使用清晰紫色描边，不能只依赖颜色变化。
- 当前路由对应的一级和子级菜单保持激活状态。
- 下载、OpenLab 等占位入口可以正常进入状态页，但不能触发真实业务操作。

## 响应式

### 桌面

- 宽度大于 1180px 时展示完整一级导航和 Mega Menu。
- 顶栏不得横向溢出；较窄桌面可以缩小一级菜单间距，但不隐藏入口。

### 平板与手机

- 宽度不大于 1180px 时切换为全屏导航抽屉。
- 一级栏目按折叠组展示，二级目录完整保留。
- 抽屉底部固定“登录 / 进入控制台”。
- Console 与 CMS 的固定侧栏切换为左侧抽屉。
- 品牌副标题可在窄屏隐藏，品牌主标题必须保留。

## 组件与配置边界

菜单数据统一放在 `apps/web/src/config/navigation.ts`，包含：

- `portalNavigation`
- `consoleNavigation`
- `adminNavigation`
- `footerNavigation`
- 菜单项的 `label`、`href`、`description`、`status`、`children` 和可选 `permission`

UI 层提供以下独立组件：

- `PortalHeader`
- `MegaMenu`
- `MobileNavigation`
- `SidebarNavigation`
- `SiteFooter`
- `NavigationStatus`

`AppShell` 根据公开门户、`/console` 和 `/admin` 三种路由边界选择相应外壳。Console 和 CMS 不直接渲染公开门户的完整导航和页脚。

配置项地址必须与 `routeRegistry` 一致。没有独立页面的二级目录先指向所属页面的稳定锚点，不创建死链接。

## 路由补齐范围

现有公开门户和 Console 的一级页面已基本覆盖。菜单实施时需要为 CMS 中尚未登记的结构补充占位路由：

- 文档管理
- OpenLab 申请审核
- License 管理
- 工单管理
- 数据统计

这些页面只实现结构、状态和接口边界，不实现外部业务。

## 测试与验收

1. 配置测试验证 PRD 要求的一级和二级菜单全部存在，地址没有重复或空值。
2. 公开导航测试验证八个一级入口、登录入口和当前路由状态。
3. Mega Menu 测试验证打开、切换、外部点击关闭、Esc 关闭和键盘焦点。
4. 移动端测试验证全部分组、登录入口、44px 点击区域和无横向溢出。
5. Console 与 CMS 测试验证侧栏分组、占位标签和权限过滤。
6. 浏览器验证 1440×1000、1024×768 和 390×844 三个视口。
7. 浏览器控制台必须无错误、资源 404 和交互警告。
8. 运行单元测试、类型检查、Lint、格式检查和生产构建。

## 不在本次范围

- 不开发 Product 页面内容。
- 不实现真实 License、下载、OpenLab、工单、订单、账单或数据统计能力。
- 不接入真实用户会话、SSO、CRM、邮件、短信和外部社群。
- 不生成虚假客户、下载量、申请量、转化率或授权数据。
- 不改变现有首页主体内容和用户提供的产品截图。

## 设计决定

- 采用方案 A“精简顶栏 + Mega Menu”。
- 不采用双层顶栏，避免压缩首页首屏。
- 不采用“核心入口 + 全站抽屉”，避免隐藏下载、OpenLab、资讯等 PRD 高频入口。
- 独立部署中心一期归入文档导航，待后续 PRD 明确后再决定是否拆成一级模块。
