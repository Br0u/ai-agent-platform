# 首页浅玻璃视觉设计

## 目标

在不改变现有首页文案、导航、模块顺序和链接的前提下，把首页统一为“浅玻璃 · 企业科技”视觉：冷白蓝页面底色、低饱和蓝靛烟雾、分层毛玻璃模块，以及克制且可中断的交互反馈。

## 范围

改动集中在首页组件与首页样式：

- `apps/web/src/components/home-sections.tsx`
- `apps/web/src/components/home.css`
- 必要时补充 `apps/web/src/app/page.test.tsx` 的稳定选择器断言

不修改共享 `AppShell`、导航、其它路由样式和当前工作区已有的未提交业务改动。

## 视觉系统

页面底色使用接近 `#f5f7ff` 的冷白蓝。首页增加独立的背景层，包含 3 个低饱和蓝靛径向渐变光斑；光斑只做极慢的 `transform` 漂移，不能影响文字对比度或模块点击区域。Hero 背景网格保留，但线条降低到低对比度。

浅色模块统一使用半透明白色表面、低对比度边框、`backdrop-filter: blur(18px)` 和柔和蓝靛阴影，并保留不支持 backdrop blur 时的实色回退。Hero 的文案与产品截图各自作为玻璃面；能力栏四项独立成轻玻璃单元；流程、企业证明、解决方案、资源区使用统一的玻璃容器承载现有列表。深色证明区和结尾 CTA 使用深靛半透明玻璃面与低亮度光晕，形成页面节奏。

## 组件与交互

不改变现有内容结构。只增加必要的背景/容器 class 和视觉层。

- 首次进入：模块使用 `opacity + translateY(8px)` 错峰出现，间隔 35–55ms；不从 `scale(0)` 开始。
- hover：仅改变 `transform`、边框高光和阴影，使用自定义 ease-out，时长 160–240ms。
- press：按钮和资源链接保留轻微 `scale(.98)` 反馈。
- 只动画 `transform`、`opacity` 和颜色/阴影；不用 `transition: all`。
- `prefers-reduced-motion: reduce` 下取消漂移、错峰和位移动画，仅保留透明度/颜色变化。
- 保留现有 `:focus-visible` 轮廓，并确保轮廓在浅色和深色玻璃面上均可识别。

## 响应式

桌面端维持当前双列 Hero 与内容网格；平板和移动端按单列排列。烟雾层在小屏降低尺寸与不透明度，产品截图保持原始比例，玻璃容器不得产生横向溢出。能力栏移动端仍为两列，正文列表保持现有移动端布局。

## 验证

实现后依次验证：

1. 首页相关 Vitest 断言保持通过，现有文案、链接和截图 alt 不变。
2. Web typecheck、lint 通过。
3. 浏览器检查桌面与移动宽度：Hero 层级、模块可读性、截图比例、模块间距、hover/active/focus 和横向溢出。
4. 在 reduced-motion 模式下确认没有背景漂移、错峰位移和入口动画。
